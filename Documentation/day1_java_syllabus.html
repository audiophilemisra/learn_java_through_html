Packages & Directory Structure

A package declaration is a namespace. The JVM requires the filesystem to mirror the package exactly. 

// File: src/com/example/day1/Calculator.java
package com.example.day1;

public class Calculator {
    public static void main(String[] args) {
        System.out.println("Compiled manually!");

}
The Mirror Rule

package com.example.day1; must live in src/com/example/day1/Calculator.java. Any mismatch produces NoClassDefFoundError at runtime.

Methods & Pass-by-Value Semantics

Java is strictly pass-by-value. For objects, the value passed is the reference copy, not the object itself. The caller's reference cannot be reassigned inside the method. 

public class Demo {
    void swapFail(StringBuilder a, StringBuilder b) {
        StringBuilder temp = a;
        a = b;        // Only local copies are reassigned
        b = temp;     // Caller sees no change
    }

    void mutateSuccess(StringBuilder sb) {
        sb.append(" mutated");  // Caller sees this — object state changed
    }
}
Method Overloading & Variable Scope

Same name, different parameter lists (type, count, or order). Return type alone is not sufficient for overloading.

Scope: local (inside a method), instance (per object, non-static field), class (shared, static field). Local shadows instance if names collide — use this.field to disambiguate.

The var Keyword (Java 10+)

Local variable type inference. The compiler infers the type from the right-hand side initializer. Improves readability when the type is obvious from the right-hand side. 

var list = new ArrayList<String>();  // Inferred as ArrayList<String>
var stream = list.stream();           // Inferred as Stream<String>

// ILLEGAL:
// var x;                    // No initializer
// var y = null;             // Null has no type
// public var field = 5;     // Not for fields
// void method(var param) {} // Not for parameters

Hour 3
Classes, Objects & Constructors
Blueprint vs. instance. Encapsulation and access control.
this Keyword & Constructor Chaining

this refers to the current object. Use it to disambiguate fields from parameters, and to chain constructors within the same class.

  • Field disambiguation: this.model = model; — this.model is the field, model is the parameter.
  • Constructor chaining: this(model, 2024); — calls another constructor in the same class. Must be the first statement.
Class Anatomy
public class Car {
    // Fields (instance variables)
    private String model;
    private int year;

    // Constructor
    public Car(String model, int year) {
        this.model = model;   // this distinguishes field from parameter
        this.year = year;
    }

    // Constructor chaining
    public Car(String model) {
        this(model, 2024);  // Delegates to 2-arg constructor
    }

    // Behavior
    public void drive() {
        System.out.println(model + " is driving.");
    }
}
Encapsulation: Why Private Fields + Public Getters/Setters
  • Validation: A setter can reject null or negative values.
  • Immutability: Omit the setter; the field becomes read-only from outside.
  • Refactoring freedom: Change internal representation without breaking callers.
public class BankAccount {
    private double balance;

    public void deposit(double amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("Amount must be positive");
        }
        this.balance += amount;
    }

    public double getBalance() {
        return balance;
    }
    // No setter — balance can only change via deposit/withdraw
}

Hour 5
Object Contracts — equals(), hashCode(), toString()
Fix broken collections. Understand the mathematical contracts.
Why This Matters

HashMap and HashSet are functionally broken without correct equals() and hashCode(). Two logically equal objects will appear as duplicates, or lookups will fail to find existing keys.

The Contracts
  • equals(): Reflexive, symmetric, transitive, consistent. Compare meaningful fields, not memory address.
  • hashCode(): If a.equals(b), then a.hashCode() == b.hashCode() must hold. The reverse is not required but desirable.
  • toString(): Human-readable debugging output. Essential for log files and debugger inspection.
public class Car extends Vehicle {
    private String licensePlate;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Car car = (Car) o;
        return year == car.year &&
               Objects.equals(brand, car.brand) &&
               Objects.equals(licensePlate, car.licensePlate);
    }

    @Override
    public int hashCode() {
        return Objects.hash(brand, year, licensePlate);
    }

    @Override
    public String toString() {
        return "Car{" + "brand='" + brand + '\'' +
               ", year=" + year +
               ", plate='" + licensePlate + '\'' + '}';
    }
}
Critical Pitfall: Mutable Fields in hashCode

Never use mutable fields in hashCode() if the object will be used as a HashMap key. Changing the field after insertion breaks the bucket location, making the key unfindable.

Practice: Prove It Works
  1. Implement equals(), hashCode(), and toString() on your Car class.
  2. Create two Car objects with identical field values.
  3. Add both to a HashSet<Vehicle>. Assert the set size is 1 (duplicates rejected).
  4. Use the same object as a HashMap key. Put a value. Look it up with the second equal object. Value must be found.

Inner Classes & Static Nested Classes
  • Inner class: Defined inside another class. Has an implicit reference to the enclosing instance. Can access private members of the outer class.
  • Static nested class: Defined inside another class with the static keyword. No reference to the enclosing instance. Behaves like a top-level class that happens to be nested for organization.
public class Outer {
    private int x = 10;

    // Inner class — tied to an Outer instance
    class Inner {
        void show() {
            System.out.println(x); // Can access outer private
        }
    }

    // Static nested class — no outer instance needed
    static class Nested {
        void show() {
            // Cannot access x directly — no enclosing instance
            System.out.println("Static nested");
        }
    }
}
Java Records (Java 16+)

Immutable data carriers. Auto-generates accessor methods (e.g., name(), not getName()), constructor, equals(), hashCode(), and toString(). 

public record Employee(String name, double salary, String dept) {
    // Compact constructor for validation
    public Employee {
        if (salary < 0) {
            throw new IllegalArgumentException("Salary cannot be negative");
        }
        if (name == null || name.isBlank()) {
            throw new IllegalArgumentException("Name required");
        }
    }
}

// Usage
var emp = new Employee("Alice", 85000.0, "Engineering");
System.out.println(emp.name());     // Accessor, not getName()
System.out.println(emp);          // Auto toString
Record Restrictions
  • Cannot extend any class (implicitly extends java.lang.Record).
  • Can implement interfaces.
  • Components are final — no setters.
  • Cannot declare instance fields outside the record header (but static fields are allowed).
Sealed Classes (Java 17+ — Brief Exposure)

Restricts which classes can extend. Natural companion to records and pattern matching. Know they exist. 

public sealed class Shape permits Circle, Square, Rectangle {
    public abstract double area();
}

public final class Circle extends Shape { ... }
public final class Square extends Shape { ... }
public non-sealed class Rectangle extends Shape { ... }

Hour 9
Strings, == vs .equals() & Wrappers
Avoid the most common Java bugs: reference comparison and autoboxing traps.
String Immutability & The String Pool

Strings are immutable. The JVM maintains a String Pool in the heap. Literal strings are interned automatically. 

String a = "hello";           // Literal → pooled
String b = "hello";           // Reuses pool reference
String c = new String("hello"); // New heap object, NOT pooled

System.out.println(a == b);       // true (same pool reference)
System.out.println(a == c);       // false (different objects)
System.out.println(a.equals(c));  // true (same content)
The Golden Rule

Always use .equals() for content comparison. Use == only for primitive values or when you explicitly want reference identity (rare).

StringBuilder vs. StringBuffer

Use StringBuilder unless you specifically need thread-safety — which in modern Java you would handle with other mechanisms anyway. 

StringBuilder sb = new StringBuilder();

    sb.append(i).append(",");

Wrapper Classes & The Autoboxing Trap

Wrappers cache values from -128 to 127. Outside that range, == compares different heap objects. 

Integer a = 127;    // Cached
Integer b = 127;    // Same cached object
System.out.println(a == b);      // true

Integer c = 128;    // New object
Integer d = 128;    // Another new object
System.out.println(c == d);      // false!
System.out.println(c.equals(d)); // true (always safe)

Hour 12
PECS, Wildcards & Bounds
Read from producers. Write to consumers. Master variance.
The PECS Principle

Producer Extends, Consumer Super. This mnemonic determines which wildcard to use when reading from or writing to a generic collection.

WildcardMeaningUse When
? extends TUnknown subtype of TReading values (Producer)
? super TUnknown supertype of TWriting values (Consumer)
Why List<Integer> is not a List<Object>

Java generics are invariant. If List<Integer> were a List<Object>, you could insert a String into what was promised to be a list of integers — breaking type safety. 

// Producer — reads from src
public static <T> void copy(List<? extends T> src, List<? super T> dst) {
    for (T item : src) {
        dst.add(item);  // Safe: dst accepts T or any supertype of T
    }
}

// Usage
List<Integer> ints = List.of(1, 2, 3);
List<Number> nums = new ArrayList<>();
copy(ints, nums);  // Integer extends Number
Swapping Bounds Breaks Compilation
// If you mistakenly write:
public static <T> void badCopy(List<? super T> src, List<? extends T> dst) {
    for (T item : src) { ... }  // Compile error: could be ANY supertype of T
    dst.add(item);               // Compile error: could be ANY subtype of T
}

Exception Handling & File I/O
Fail fast. Recover gracefully. Never swallow exceptions silently. Modern file handling with NIO.2.
Exception Hierarchy
Throwable
├── Error              // Unrecoverable: OutOfMemoryError, StackOverflowError
│                      // Do NOT catch these.
└── Exception
    ├── RuntimeException // Unchecked: NullPointerException, IllegalArgumentException
    │                  // Compiler does not force handling.
    └── [Other]        // Checked: IOException, SQLException
                       // Compiler forces try/catch or throws declaration.
Modern Exception Handling
// Multi-catch (Java 7+)
try {
    // risky operation
} catch (IOException | SQLException e) {
    System.err.println("Data access failed: " + e.getMessage());
    throw new AppException("Operation failed", e);  // Wrap and rethrow
}

// try-with-resources (AutoCloseable)
try (BufferedReader br = new BufferedReader(new FileReader("file.txt"))) {
    return br.readLine();
} catch (IOException e) {
    throw new UncheckedIOException(e);
}

// Custom exception
public class MathException extends Exception {
    public MathException(String msg) { super(msg); }
}
Anti-Pattern: Catch-and-Swallow
try {
    // operation
} catch (Exception e) {
    // EMPTY — never do this. The program continues in a corrupt state.
}

Never catch Exception broadly unless logging and rethrowing. Fail fast.

File I/O — Classic & NIO.2
// Classic I/O (legacy awareness — prefer NIO.2)
try (BufferedReader br = new BufferedReader(new FileReader("notes.txt"))) {
    String line;
    while ((line = br.readLine()) != null) {
        System.out.println(line);
    }
}
import java.nio.file.*;

// NIO.2 (modern, preferred API)
Path p = Paths.get("data", "notes.txt");  // Cross-platform

// One-liner reads
List<String> lines = Files.readAllLines(p);
String content = Files.readString(p);       // Java 11+

// One-liner writes
Files.writeString(p, "Hello, NIO.2!", StandardOpenOption.APPEND);

// Directory checks
boolean exists = Files.exists(p);
boolean isDir  = Files.isDirectory(p);

JVM Memory Model Deep Dive

Garbage Collection Basics
  • Eden: New objects allocated here.
  • Survivor Spaces (S0, S1): Objects surviving Eden GC get copied here.
  • Old Generation: Long-lived objects promoted from Survivor.
  • Eligibility: An object is eligible when no references point to it. Setting a reference to null or reassigning it can make the old object eligible.
Object heavy = new Object();
// ... use heavy ...
heavy = null;  // Old object is now eligible for GC (but not immediately collected)
Compilation Pipeline
  1. .java source file
  2. javac → bytecode .class files (platform-independent)
  3. JVM ClassLoader loads .class into Metaspace
  4. Interpreter executes bytecode initially
  5. JIT Compiler (HotSpot) compiles frequently executed bytecode to native machine code for performance
